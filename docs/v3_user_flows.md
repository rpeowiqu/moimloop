# 반복 모임 운영 서비스, Moimloop

## V3 유저 플로우

#### Flow V3-A. 시스템이 알림과 후속 자동화를 수행한다

**목표**

운영자가 직접 확인하지 않아도 시스템이 회차 상태에 따라 자동으로 알림을 보냄

**흐름**

1. 시스템이 회차 상태를 감지한다.
2. 시스템이 멤버에게 알림 또는 리마인드를 보낸다.
3. 멤버가 해당 알림을 통해 다시 회차에 진입해 응답한다.

**결과**

- 운영자의 수동 리마인드 부담이 줄어든다.

#### Flow V3-B. 공개 모임 생성과 탐색이 가능해진다

**목표**

제품이 고정 멤버 운영 도구를 넘어 공개 참여형 모임 서비스로 확장

**흐름**

1. 운영자는 모임을 공개 설정으로 연다.
2. 사용자는 공개 모임을 탐색한다.
3. 사용자는 모임에 참여하거나 신청한다.

**결과**

- 고정 멤버 운영 도구를 넘어 탐색 서비스 흐름이 추가된다.

#### Flow V3-C. 채팅과 실시간 협업이 회차 안에서 일어난다

**목표**

운영자와 멤버가 회차 안에서 직접 대화를 나눌 수 있음

**흐름**

1. 사용자는 회차 안에서 메시지를 남긴다.
2. 다른 사용자가 실시간으로 확인하고 답한다.
3. 논의 결과가 회차 진행에 반영된다.

**결과**

- 회차 운영과 대화가 한 제품 안으로 들어온다.

#### Flow V3-D. 운영자가 통계와 리포트를 확인한다

**목표**

운영자가 누적된 회차 데이터를 바탕으로 모임 운영 패턴을 볼 수 있다.

**흐름**

1. 운영자는 참여율, 응답 패턴, 회차 수 같은 지표를 본다.
2. 운영자는 이를 바탕으로 모임 운영 방식을 조정한다.

**결과**

- 누적 데이터를 기반으로 한 운영 인사이트가 생긴다.
