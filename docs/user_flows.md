# 반복 모임 운영 서비스, Moimloop

## 0. 문서 목적

Moimloop 프로젝트의 사용자 흐름을 V1 / V2 / V3 기준으로 정리한 문서

## 1. V1 핵심 사용자 흐름

V1의 핵심 루프: `모임 생성 또는 선택 -> 회차 생성 -> 멤버 응답 -> 운영자 확인/확정 -> 회차 종료 -> 다음 액션 기록 -> 다음 회차 시작 시 참고`

### V1 Core Loop Diagram

```mermaid
flowchart LR
    subgraph Operator["운영자"]
        O1["모임 생성 또는 모임 선택"]
        O2["회차 생성"]
        O3["일시, 확정 장소, 준비 항목 입력"]
        O4["응답 가능한 상태로 열기"]
        O5["현재 회차 상태 확인"]
        O6{"미응답 또는 미배정이 남았나?"}
        O7["정보 보완 후 계속 수집"]
        O8["회차 확정"]
        O9["회차 종료"]
        O10["종료 메모와 다음 액션 기록"]
        O11["다음 회차 시작 시 직전 기록 참고"]
    end

    subgraph Member["멤버"]
        M1["링크로 회차 진입"]
        M2["참석 상태 선택"]
        M3["준비 항목 확인 및 담당 표시"]
        M4["응답 저장 또는 수정"]
    end

    O1 --> O2 --> O3 --> O4 --> M1
    M1 --> M2 --> M3 --> M4 --> O5
    O5 --> O6
    O6 -- "예" --> O7 --> O5
    O6 -- "아니오" --> O8 --> O9 --> O10 --> O11 --> O2
```

## 2. V2 확장 흐름

V2는 V1 핵심 루프에서 운영자의 반복 업무를 더 줄이는 것을 목적으로 함

### V2 Expansion Diagram

```mermaid
flowchart LR
    A["기본 회차 운영"] --> B["이전 회차 히스토리 확인"]
    B --> C["기존 회차 복제 또는 템플릿 사용"]
    C --> D["여러 모임 전환 관리"]
    D --> E["미응답자 간단 리마인드"]
    E --> F["필요 시 정산 메모 추가"]
    F --> G["다음 회차 운영 시간 단축"]
```

[V2 상세 유저 플로우](./v2_user_flows.md)

## 3. V3 확장 흐름

V3는 서비스의 확장을 목적으로 하며 자동화, 채팅과 실시간 협업, 누적 통계와 리포트 등을 목적으로 한다.

### V3 Expansion Diagram

```mermaid
flowchart LR
    A["기본 회차 운영"] --> B["시스템 알림과 자동 리마인드"]
    A --> C["공개 모임 생성과 탐색"]
    A --> D["채팅과 실시간 협업"]
    A --> E["누적 통계와 리포트"]
```
