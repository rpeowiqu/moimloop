# 반복 모임 운영 서비스, Moimloop

## V2 유저 플로우

#### Flow V2-A. 운영자가 지난 회차 히스토리를 확인한다

**목표**

운영자가 직전 회차를 넘어서 더 이전 회차의 기록도 다시 찾아볼 수 있음

**흐름**

1. 운영자는 특정 모임의 회차 기록을 연다.
2. 운영자는 종료된 회차 목록을 본다.
3. 운영자는 특정 회차를 열어 메모와 준비 내용을 확인한다.

**결과**

- 과거 회차를 다시 참고할 수 있다.

#### Flow V2-B. 운영자가 여러 모임을 전환해 관리한다

**목표**

운영자가 둘 이상의 스터디 또는 반복 모임을 전환하며 관리

**흐름**

1. 운영자는 관리 중인 모임 목록을 본다.
2. 운영자는 특정 모임을 선택해 현재 회차 상태를 확인한다.
3. 운영자는 모임 간을 전환하며 회차 준비를 계속한다.

**결과**

- 하나의 계정 또는 운영 경험 안에서 여러 모임을 다룰 수 있다.

#### Flow V2-C. 운영자가 이전 회차를 복제해 다음 회차를 만든다

**목표**

운영자가 반복 입력을 줄여 더 빠르게 다음 회차를 만들기

**흐름**

1. 운영자는 기존 회차를 기준으로 새 회차 생성을 시작한다.
2. 운영자는 일시와 필요한 일부 정보만 수정한다.
3. 운영자는 회차를 저장하고 응답 가능한 상태로 연다.

**결과**

- 다음 회차가 생성된다.
- 다음 회차 생성 시간이 줄어든다.

#### Flow V2-D. 운영자가 미응답자에게 간단 리마인드를 보낸다

**목표**

운영자가 별도 채팅방 정리 없이 미응답 상태를 다시 확인시킴

**흐름**

1. 운영자는 현재 회차에서 미응답자를 확인한다.
2. 운영자는 간단 리마인드 동작을 선택한다.
3. 미응답자가 다시 회차 링크를 확인하고 응답한다.

**결과**

- 미응답 유저들에게 알림이 간다.

#### Flow V2-E. 운영자가 정산 메모를 남긴다

**목표**

운영자가 회차별 비용이나 정산 필요 여부를 가볍게 메모

**흐름**

1. 운영자는 회차 종료 또는 회차 상세에서 비용 관련 메모를 남긴다.
2. 운영자는 이후 필요 시 회차 맥락에서 해당 메모를 다시 확인한다.

**결과**

- 회차별 비용 메모가 남는다.
