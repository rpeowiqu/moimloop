# moimloop

> 소규모 고정 멤버 모임의 “다음 회차 운영”을 가볍게 관리하는 서비스, 모임루프

## 1. 프로젝트 소개

이 프로젝트는 친구 정기모임, 스터디, 취미 모임처럼
반복적으로 열리는 소규모 모임의 운영 부담을 줄이기 위한 서비스입니다.

기존에는 채팅방, 설문, 캘린더, 메모에 흩어져 처리하던 회차 운영 정보를
한 곳에서 정리해 운영자가 다음 회차를 더 쉽게 준비할 수 있게 만드는 것이 목표입니다.

## 2. 해결하려는 문제

소규모 반복 모임의 운영자는 매 회차마다 아래를 반복해서 정리해야 합니다.

- 참석 여부 확인
- 준비물 / 역할 분담 정리
- 회차 상태 확정
- 지난 회차 메모와 다음 액션 연결

기존 도구 조합만으로는 “현재 회차의 운영 상태”를 한눈에 보기 어렵고,
운영자 한 명에게 수동 정리 부담이 몰리는 문제가 있습니다.

## 3. 핵심 방향

이 서비스는 다음에 집중합니다.

- “모임 전체”보다 “현재 회차 운영”
- 채팅 대체가 아니라 운영 상태 정리
- 정산 중심이 아니라 운영 중심
- 개인 프로젝트 범위에서 완성 가능한 경량 제품

## 4. 타깃 사용자

1차 타깃은 아래 사용자입니다.

- 스터디 운영자
- 취미 모임 운영자
- 고정 멤버 4~10명 규모의 반복 모임 리더 / 총무

## 5. V1 핵심 기능

- 모임 생성
- 회차 생성 / 수정
- 회차 상태 관리 (준비 중 / 확정 / 종료)
- 참석 응답 (참석 / 불참 / 미정)
- 준비물 / 역할 분담
- 미응답 / 미배정 항목 확인
- 회차 종료 후 다음 액션 메모
- 지난 회차 히스토리 확인
